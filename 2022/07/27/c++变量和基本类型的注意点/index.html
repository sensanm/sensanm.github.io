c++变量和基本类型的注意点

一.内置和基本类型

  • 当无符号数与有符号数相加时 不是简单的算术相加而是先将有符号数先转换为无符号数再相加

    • ```c++
      unsigned i=10;
      int a=-20;
      std::cout<<i+a<<std::endl;
      std::cout<<i-a<<std::endl;

二.变量

  • c++ 的初始化不等同于赋值，两者在底层有着不同的操作空间

    • 在用列表初始化的时候如果存在初始值丢失风险的时候编译器将会报错。

    • 如果再定义变量时没有初始化我们将会默认初始化。

      定义在函数之外的变量将被初始化为0

      定义再函数之内的内置类型将不被初始化，如果拷贝或者复制将会引发错误

  • 变量只能被定义一次但可以被多次声明

    • 任何显示的声明都会变成定义，在函数内部如果试图初始化一个由extern 关键字的子标记变量将会引发错误
    • 记住再多个文件共同用到的变量只能被顶定义一次

  • 作用域

    • 绝大多数的作用域都是在花括号进行分割的

    • 还可以使用作用域操作符：：来访问不同作用域的值

      *

    • &以及*是修饰符不算类型

  • const 限定符

    • const 对象必须进行初始化，因为他的值不能发生改变

    • 默认条件下，const 对象只有再文件内有效

    • constexper 和常量表达式：

      • 他的值一般在编译的时候就得到了。

      常量表达式比较难判断，一般用constexpr 来判断是否是常量表达式。如果要确认一个变量是常量表达式，呢么就帮他声明成constexpr。

      常量表达式的左值和右值要是常量表达式

      • 当用constexpr 来声明一个指针的时候，指针必须是 0 或者指向一个固定的位置，也就说明指向的值必须具有全局作用域。
      • constexpr 来声明指针时只对指针有效，对其所指的对象无效，也就是说constexpr 只能用于来描述顶层。
